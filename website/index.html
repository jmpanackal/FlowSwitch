FAQ

Quick answers before you install.

Install, privacy, and Windows prompts—straight answers, no marketing fog.

What is a workspace profile?

A workspace profile captures your active apps, monitor placement, launch order, and browser context. Instead of rebuilding your setup manually, you apply one profile and FlowSwitch restores the state.

Does FlowSwitch work with ultrawide monitors?

Yes. FlowSwitch captures monitor geometry and window bounds, so ultrawide setups can be restored precisely as long as the same display is available when you switch.

Does it support multiple monitors with different resolutions?

Yes. Mixed-DPI and mixed-resolution monitor arrangements are a core use case, and placement is stored per display so windows return to the intended screen.

Can I use it without admin rights?

Most profile capture and launch workflows run in standard user mode. Some enterprise environments may apply restrictions on launching certain apps or reading window metadata.

How is this different from Windows virtual desktops?

Virtual desktops separate window groups, but they do not rebuild full workflows on demand. FlowSwitch restores app launch order, monitor positions, and browser context in one action.

Is my layout data stored in the cloud?

FlowSwitch is local-first. Profile data stays on your machine by default, and cloud storage is not required for workspace switching.

Does FlowSwitch send my window layouts or apps list to your servers?

No. Profiles, captured layouts, and launch metadata are written to local disk under your Windows user profile. Profile switching does not upload your window layouts or app list to FlowSwitch servers. This marketing site may fetch release metadata over HTTPS (for example latest.json) so download buttons can point at the right installer on GitHub.

Why does Windows SmartScreen or “Windows protected your PC” appear when I install?

The installer is not yet signed with an expensive commercial code-signing certificate, so Microsoft may show a reputation warning for new downloads. That dialog is about trust of the file, not a virus verdict. Choose “More info” (or similar), then “Run anyway,” if you are comfortable installing software from this site and GitHub Releases. Signing the app reduces that friction and is on the roadmap for distribution.
